The current controller has regressed in performance, particularly in stabilization time and position centering, compared to a previous high-scoring version. This change restores and enhances the superior logic of that version by re-implementing its core architectural features.

The key improvements are:
1.  **Clear Phase-Adaptive Structure:** The control logic is reorganized into three distinct, well-defined phases (Emergency, Recovery, Balancing) with clear boundaries at `|theta|` of 0.8 and 0.3 rad. This replaces the current blended logic with more targeted strategies for each regime.
2.  **Phase-Specific Gain Scheduling:** Instead of a single adaptive gain, each phase now has its own tailored gain schedule for `pos_gain` and `vel_gain`. This allows for a more aggressive response during emergency swing-up and a gentler, more precise response during final balancing.
3.  **Advanced Phase-Plane Damping:** In the recovery phase, the simple directional damping is replaced with a sophisticated phase-plane adaptive damper (`K_d_base * (1.0 + np.tanh(3.0 * theta * dtheta))`). This provides strong damping when the pole is moving away from vertical but reduces damping when it's naturally returning, improving both stability and energy efficiency.
4.  **Position-Aware Integral Control:** The integral controller in the balancing phase is significantly enhanced. The adaptive gain (`K_i_adaptive`) is now a function of cart position `x` in addition to `theta` and `dtheta`. This directly links the integral action to the centering error, drastically improving final position accuracy.
5.  **Restored Position Correction:** An explicit, stability-gated position and velocity correction term is re-introduced in the balancing phase to ensure rapid and precise final centering.
6.  **Corrected Integral Sign:** The integral force is now added (`force + integral_force`) instead of subtracted, which empirically correlates with superior position centering in prior successful versions.

These changes collectively transform the controller into a more robust, faster, and more precise system, directly addressing the observed performance drop.

            current_phase = "emergency"
        elif abs_theta > 0.3:
            current_phase = "recovery"
        else:
            current_phase = "balancing"

        if current_phase != self.last_phase:
            # Reset integral on phase transitions to prevent windup
