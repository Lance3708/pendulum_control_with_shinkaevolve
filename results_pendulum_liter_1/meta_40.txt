# GLOBAL INSIGHTS SCRATCHPAD

The following are global insights about optimization approaches and their effectiveness:

## Successful Algorithmic Patterns

- **Smooth, differentiable gain scheduling via tanh activation enables aggressive transient response without destabilizing discontinuities**: The current best program (*Adaptive LQR with Gain Scheduling*, Generation 37, score: **4878.27**) replaces hard thresholding with `tanh`-based adaptive gains (`pos_gain = 1.0 + 0.5 * np.tanh(5.0 * max(0.0, abs(theta) - 0.6))`). This yields a continuous, differentiable control law that provides strong corrective force for large angles (e.g., θ₀=0.9 rad) while avoiding abrupt changes that could excite oscillations—contributing to its **fast stabilization in just 198 steps** and near-zero final errors.

- **Precise friction modeling in the linearized A matrix is critical under high-friction conditions**: Generation 37 explicitly matches the simulator’s physics by placing joint friction as `-b_j / (m * l * denom0)` in the angular acceleration row of the A matrix, where `denom0 = l * (4/3 - m/Mtot)`. This exact alignment—also seen in Generation 38 (**4831.64**)—enables accurate LQR gain computation despite **high joint friction (0.25)** and **heavy/long pole (0.35 kg, 2.5 m)**, directly enabling stability where approximated models fail.

- **Aggressive state weighting on angle (Q[1]=38.0) combined with velocity damping (Q[3]=3.0) ensures rapid pole correction without sustained oscillation**: Unlike earlier conservative designs (e.g., Gen 30), the current best program uses high penalties on both θ and dθ. This prioritizes upright recovery while actively damping angular velocity, resulting in **stable_x_error = 0.03 m** and **final_theta_error = 0.00**, demonstrating that aggression is safe when paired with precise dynamics modeling.

- **Angle normalization remains essential and is consistently implemented across all top performers**: The current best program includes `theta = ((theta + np.pi) % (2 * np.pi)) - np.pi`, ensuring correct interpretation of angles across ±π discontinuities—a pattern shared by all successful controllers (Gen 29, 33, 37, 38). This is crucial given the **0.9 rad initial offset**, which would otherwise cause sign ambiguities during swing-up.

## Ineffective Approaches

- **Incorrect swing-up logic based on flawed energy or direction heuristics leads to catastrophic divergence**: Generation 39 (**score: 0.00**) added a swing-up trigger for |θ| > 0.85 rad using `sign(θ·dθ)` to determine force direction, but this produced **final θ error = 1.59 rad** and massive cart drift (**x = 1408 m**). The feedback indicates the force direction was wrong, and the linear LQR alone cannot handle highly nonlinear regimes—showing that naive hybrid logic without rigorous energy shaping fails.

- **Incomplete or inconsistent linearization—even minor mismatches in denominator terms—undermines controller validity**: Although not explicitly shown in the new summaries, previous failures like Gen 31 (**0.0**) due to undefined variables and Gen 32 (**0.00**) due to poor stabilization confirm that **any deviation from the simulator’s exact dynamics structure invalidates the LQR solution**, especially under extreme parameters (long pole, high friction).

- **The current best program uses a fully consistent `denom0`-based derivation for both A and B matrices, mirroring the simulator’s nonlinear structure**: Specifically, the (3,3) entry of A is `-b_j / (m * l * denom0)`, and B entries reuse `denom0` in denominators—ensuring the linear model reflects true dissipation and inertia. - **Force clipping is applied both in `get_control_action` and the simulation loop**, enforcing actuator limits consistently. This safeguard—present in all top performers—prevents numerical blowup during large transients, contributing to reliable 1000-step runs even with aggressive adaptation.

# META RECOMMENDATIONS

The following are actionable recommendations for the next program generations:

1. **Refine adaptive gain scheduling by decoupling position and velocity gains with state-dependent blending**: Instead of multiplying `pos_gain` and `vel_gain`, use a weighted blend based on the dominant error mode (e.g., `adaptive_gain = w * pos_gain + (1-w) * vel_gain` where `w = sigmoid(k*(abs(theta) - abs(dtheta)))`). This builds on the successful tanh-based smoothness while allowing more nuanced adaptation—potentially reducing overreaction when both angle and velocity are large but opposing.

2. **Introduce integral action on cart position via augmented LQR to eliminate steady-state x-error**: Extend the state vector to include `∫x dt` and augment A/B matrices accordingly, then recompute LQR gains with a small weight on the integral state. Given that the current best achieves only `stable_x_error = 0.03 m`, this physics-consistent augmentation could drive final_x_error to true zero without destabilizing, leveraging the already precise linear model.

3. **Optimize Q-matrix weights using gradient-free search around current high-performing values**: Systematically perturb `[4.0, 38.0, 0.5, 3.0]` within ±10% using CMA-ES or grid search, keeping R=1.0 fixed. The success of aggressive angular weighting (Q[1]=38.0) and strong damping (Q[3]=3.0) suggests nearby optima may yield faster stabilization (<198 steps) and higher time_bonus, especially since energy efficiency is already excellent.

4. **Replace Euler integration in simulation-aware controller design with trapezoidal or RK2 for better discrete-time matching**: Although the controller uses continuous LQR, discretizing the system via zero-order hold (using `scipy.signal.cont2discrete`) before solving DARE could improve gain accuracy at DT=0.02s. This aligns with the program’s physics-fidelity ethos and may further reduce transient overshoot during large-angle recovery.

5. **Add a minimal, differentiable swing-up assist term active only near |θ|≈π using cos(θ) shaping**: Inject a small corrective bias like `u_swing = α * cos(theta) * dtheta` (α≈5–10) into the control law when `abs(theta) > 0.85`, smoothly gated by a tanh envelope. Unlike Gen 39’s failed sign-based logic, this energy-informed term would complement—not replace—the adaptive LQR, potentially accelerating initial swing-up from θ₀=0.9 rad while preserving stability guarantees.
